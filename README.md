# Axis Tasks Platform PRD

## 1. 背景 & 目标

### 1.1 背景

- RoboVerse/METASIM 已经提供了统一的机器人仿真基础设施、任务配置格式和大规模任务/轨迹数据能力。
- Axis 希望把这些能力包装成面向「普通用户」的产品入口，让非专业用户也能参与到机器人智能的训练和评估中。
- 技术团队正在探索：
    - 浏览器本地仿真（mujoco_wasm 等）；
    - WebSocket 遥操作；
    - 中央化数据存储与后端评估（ERM）等。

### 1.2 产品定位

> Axis 任务平台
> 
- 面向对象：对 AI / 机器人 / Web3 有兴趣的普通用户（非专业机器人研究员）。
- 产品形态：Web 为主（桌面优先），小屏幕移动端设备浏览器兼容

### 1.3 mvp产品目标（Must-have）

1. 用户可以：
    - 访问任务大厅，看到若干预设任务；
    - 进入某个任务详情，理解要做什么；
    - 在浏览器中遥操作机器人完成一局任务；
    - 在结束后看到清晰的结果与「贡献值」反馈。
    - 通过Privy 登录
2. 后端可以：
    - 接收任务执行的轨迹数据（状态/动作/结果）；
    - 做一版基础数据打分（简化版 ERM）；
    - 将结果映射为用户侧可见的「贡献值」。

## 2. 用户 & 场景

### 2.1 核心用户画像（Player）

- 年龄：20–35 岁。
- 背景：Web3 玩家 / AI 爱好者 / 开发者 / 学生。
- 技术能力：会用浏览器玩网页游戏即可。
- 动机：
    - 想体验「控制机器人」的新鲜感；
    - 想为前沿 AI/机器人研究做贡献；
    - 希望有一定成就感、等级/积分反馈。

### 2.2 核心使用场景

**场景 S1：新人第一次进入平台**

1. 从社交媒体/活动链接打开 Axis 任务平台。
2. 在首页看到「像打游戏一样帮机器人学习」文案。
3. 点击「开始第一个任务」，进入新手任务详情。
4. 了解任务说明后，点击「开始任务」。
5. 在浏览器中控制机器人完成（或失败）一局。
6. 在结果页看到任务结果+贡献值，愿意再玩一次或尝试新任务。

**场景 S2：已有用户日常上线刷任务**

1. 登录后进入任务大厅。
2. 按「新手友好」「难度」「用时」筛选任务。
3. 选择一个任务连续玩几局，观察自己表现。
4. 在结果页看到贡献值增加、等级进度条变化。
5. 在「我的」页面查看历史记录和总贡献

## 3. 需求范围 & 版本规划

### 3.1 v0.1 范围（MVP）

**页面级：**

1. 首页（任务大厅）
2. 任务详情页
3. 任务运行页
4. 结果页
5. 简单的「我的」页面（历史记录 + 贡献值）

**功能级：**

- 基本账号登录（邮箱/社交；钱包登陆）。
- 任务列表展示 + 筛选。
- 单任务详情展示。
- 浏览器内仿真运行（调用底层仿真接口）。
- 轨迹采集并上传。
- 后端数据打分 + 贡献值计算。
- 前端结果页展示贡献值。

### 3.2 后续版本方向

- v0.1.1：
    - 简单任务编辑器（Creator）
    - 模型回放对比（人 vs 模型）
    - 更完善的账号体系（钱包、Web3 身份）
    - 评审/标注工作台（Judge）
    - 排行榜 & 勋章体系
    - API / SDK 接入更多任务类型

## 4. 信息架构（IA）

**一级导航：**

- 任务大厅（首页）
- 我的
- 登录/个人头像

**页面关系：**

- index（任务大厅）
    
    → 点击任务卡片 → task（任务详情）
    
    → 点击开始任务 → play（运行页）
    
    → 任务结束（成功/失败） → result（结果页）
    
    → 从结果页可回到 play（再玩一次）或 index（换任务）

## 5. UI 示意

### 5.1 首页 / 任务大厅

![./design/Task-Hall.png]

**模块：**

1. 顶部导航栏
    - 显示产品名 Axis Tasks
    - 导航：任务大厅 / 我的 / 登录
2. Banner
- 文案 + 副标题
- CTA 按钮「开始第一个任务」→ 打开一个预设新手任务详情
3. 左侧筛选区
    - 筛选项（静态即可）：
        - 任务类型：全部 / 抓取 / 推物体 / 排序…
        - 难度：全部 / 新手友好 / 中等 / 挑战
        - 预计用时：不限 / ≤3 分钟 / 3–5 分钟 / ≥5 分钟
4. 右侧任务列表
    - 任务卡片字段：
- 缩略图（静态占位图或简单动图）
- 标题（任务名）
- 一句话描述
- 元信息：难度、平均完成率、预计用时
- 按钮：「查看详情」→ 跳到任务详情页

### 5.2 任务详情页

![./design/Task-Detail.png]

**模块：**

1. 顶部导航 + 返回按钮
2. 标题区
    - 显示任务名称
3. 左侧预览区域
    - 占位图/动图（示意机器人完成任务）
4. 右侧任务说明区域
    - 目标描述（1–2 句）
    - 成功条件
    - 失败条件
    - 预计用时
    - 难度
    - 控制方式说明（键位/鼠标）
    - 机器人类型（例如桌面机械臂）
    - 「开始任务」按钮 → 跳到运行页
5. 底部统计/社会证明
- 已参与人数
- 平均完成率
- 适合人群（新手/有经验）

### 5.3 任务运行页

![./design/Task-Running.png]

**模块：**

1. 顶部导航
    - 左：Logo
    - 右：退出（回任务详情）、重置任务（重新开始本任务）
2. 当前任务标题条
3. 左侧仿真画面区
    - Canvas / WebGL 占位（由前端接入实际仿真）
    - 捕获用户输入（键盘、鼠标）
4. 右侧信息面板
    - 模块：
        - 任务目标：简短文本
        - HUD：
            - 剩余时间
            - 当前步数
            - 当前状态（操作中/成功/失败）
        - 操作说明：
- 键位映射（WASD / 鼠标 / 空格）
- 提示/消息区域：
    - 临终提示（剩 10 秒）
    - 失败原因（如方块掉出桌面）
5. 结束逻辑
    - 前端从仿真模块接收 success_flag 或 done_flag。
    - 若 success_flag == true → 成功
    - 若 done_flag == true & success_flag == false → 失败
    - 调用前端结束处理：
        - 结束记录（不再接收输入）
        - 序列化本局轨迹并上传后端
        - 跳转结果页

### 5.4 结果页

![./design/Result.png]

**模块：**

1. 成功/失败面板
    - 成功：✅ +「任务完成！」文案
    - 失败：❌ +「没关系，再试一次」文案
2. 任务信息
    - 任务名
    - 本局用时
    - 步数
    - 结果（成功/失败）
3. 贡献值面板
    - 贡献度星级（由后端评估结果映射）
    - 本次获得贡献值（如 +12）
    - 当前总贡献值
    - 等级（Lv1–LvN）及进度条
    - 距离下一级差多少贡献值
4. 行动按钮
    - 再玩一次本任务 → 返回运行页
    - 去看看其他任务 → 返回任务大厅
